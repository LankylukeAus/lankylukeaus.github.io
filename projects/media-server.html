🎬 Media Server — automated & VPN-gated

Jellyfin + *arr suite behind a gluetun VPN kill-switch
Jellyfin Sonarr / Radarr Prowlarr gluetun VPN ● running
Summary A self-hosted media platform — Jellyfin for streaming with the *arr automation suite (Sonarr, Radarr, Prowlarr, Jellyseerr) managing the library and requests. The interesting part is the networking: all download traffic is forced through a gluetun VPN container with a kill-switch, so if the tunnel drops, nothing routes.

01What it does & the networking

Jellyfin serves the media library to any device. Prowlarr manages indexers, Sonarr and Radarr handle TV and film automatically, and Jellyseerr gives a friendly "request a title" front end. The download client runs with its network routed entirely through the gluetun container (network_mode: service:gluetun) — so its only path to the internet is the VPN tunnel. Gluetun's firewall is configured to drop all traffic if the VPN goes down (a kill-switch), which is the real engineering lesson here: enforcing an egress path at the container-network level rather than trusting an app setting.

03Why I built it & skills shown

  • Container networking — routing one container's whole network through another and enforcing a VPN-only egress path.
  • Kill-switch / fail-closed design — traffic stops if the tunnel drops, rather than leaking.
  • Service orchestration — half a dozen interdependent services wired together with health & dependency ordering.
  • Self-hosting — replacing multiple SaaS subscriptions with one maintained stack.
